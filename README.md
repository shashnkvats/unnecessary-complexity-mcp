# Unnecessary Complexity MCP Server

A playful MCP server that turns simple ideas into absurdly over-engineered enterprise architecture output.

It exposes a set of tools that use an LLM to:
- over-design tiny tasks,
- explain fake microservices with complete seriousness,
- translate plain English into consultant-style buzzword soup, and
- estimate outrageous delivery complexity.

## Features

- Built with `FastMCP`
- Uses OpenAI chat completions under the hood
- Validates tool input with Pydantic models
- Loads secrets from `.env`
- Designed to connect directly to Claude Desktop via MCP

## Prerequisites

- Python `3.13+`
- `uv` installed
- An OpenAI API key

## Installation

From the `unnecessary-complexity` directory:

```bash
uv sync
```

Create a `.env` file:

```env
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_MODEL=gpt-5.2
```

`OPENAI_MODEL` is optional; defaults to `gpt-5.2` if not set.

## Tools and Use Cases

### 1) `overcomplexify_task`

**What it does**
- Takes a simple task (for example: "make tea")
- Produces a dead-serious enterprise architecture doc with many microservices
- Optionally includes an ASCII diagram

**Input**
- `task` (string): task to over-engineer
- `complexity_tier` (int, 1-5): how absurd the architecture should get
- `include_diagram` (bool): include ASCII architecture diagram

**Use cases**
- Demoing MCP tool calling in a fun way
- Icebreakers in engineering meetings
- Generating satire architecture docs for talks/content

---

### 2) `explain_fake_service`

**What it does**
- Takes a made-up microservice name
- Returns a highly technical, serious explanation of its responsibilities, dependencies, and failure impact

**Input**
- `service_name` (string): fake service name (for example: `TaskIngestionOrchestrator`)

**Use cases**
- Standup humor
- Architecture role-play exercises
- Generating believable fake internal service documentation

---

### 3) `buzzwordify`

**What it does**
- Rewrites plain text into maximum enterprise jargon
- Preserves rough meaning while inflating complexity and wording

**Input**
- `text` (string): plain description to transform  
  (also accepts alias `simple_description`)

**Use cases**
- Writing parody "enterprise speak"
- Creating before/after examples for communication workshops
- Generating intentionally unreadable release-note jokes

---

### 4) `estimate_complexity`

**What it does**
- Produces a fake but formal complexity/scoping document
- Includes story points, sprint estimates, ticket breakdown, dependencies, risks, and acceptance criteria

**Input**
- `task` (string): task to estimate  
  (`complexity_tier` and `include_diagram` are accepted by schema but not used here)

**Use cases**
- Satirical planning artifacts
- Team demos about estimation anti-patterns
- "How not to scope work" examples

## Running the Server Locally

From the `unnecessary-complexity` directory:

```bash
uv run python server.py
```

## Connect to Claude Desktop

Claude Desktop discovers local MCP servers from `claude_desktop_config.json`.

### 1) Open Claude Desktop config file

Typical locations:
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

### 2) Add this MCP server entry

Use absolute paths for `cwd` and your Python environment command:

```json
{
  "mcpServers": {
    "unnecessary-complexity": {
      "command": "uv",
      "args": ["run", "python", "server.py"],
      "cwd": "absolute_path",
      "env": {
        "OPENAI_API_KEY": "your_openai_api_key_here",
        "OPENAI_MODEL": "gpt-5.2"
      }
    }
  }
}
```

If you prefer keeping secrets in `.env`, you can omit the `env` block and keep `.env` in the project root.

### 3) Restart Claude Desktop

Fully quit and reopen Claude Desktop so it reloads MCP server configuration.

### 4) Verify it works

In Claude, you should see the server and tools:
- `overcomplexify_task`
- `explain_fake_service`
- `buzzwordify`
- `estimate_complexity`

## Notes

- This project is intentionally comedic and not meant for production architecture decisions.
- Tool outputs are generated by an LLM and may vary run-to-run.
